1. Introduction

1.1 Project summary

WLU Go is a web application that allows users to view, rate, and review the courses offered at Wilfrid Laurier University (WLU). It would help students get an idea of what a course is like by reading reviews and different rating parameters. 

As of now, there is no dedicated way for WLU students to get a thorough understanding of the courses offered other than through word of mouth. We want WLU Go to be a one-stop universal web application that is easily accessible and provides information on any WLU course with just a quick course code search. 

The scope of the project is a fully functional web-based application that allows Wilfrid Laurier University students access to information on every course offered at the university. The objective of WLU Go is to help students gain a better understanding of WLU courses based on the experiences of others, desired birdness (difficulty), enjoyability, and usefulness. Through the use of this application, students will have an easier time deciding which courses to take.

 

1.2 Project purpose

This document will cover all of the analysis needed for WLU Go and will describe all of its features and how they work.

The intended audience for this document will be the client (David Brown) and the project team. They will use this document to be reminded of the analysis of the project. The document will also be used as the basis for the project.

The intended audience for this web application is future and current Wilfrid Laurier University students who are looking to get some insight on the courses which they will potentially take or want to leave some knowledge about their experience with their respective course. In addition, the users of this web application are expected to have a basic understanding of how to use a modern website and internet browser. 

It will be helpful but not necessary if users have previous experience with other community-like social platforms (e.g. Reddit or Facebook). With this prior expertise, users will be comfortable using some of the features WLU Go has to offer, such as a comment system, rating network, and feedback loop.

2.2 Entity Objects

Users - Typically a student who is enrolled at Wilfrid Laurier University who wants to view or review some Laurier courses. 

Courses - Courses are all of the classes which are currently offered at Wilfrid Laurier University. 

Post - A post is an object that will only include text characters. Posts will be created by Users for all of the Group to see.

Profile - The users profile, which holds their information, such as their email address.

Page - Navigation through different pages that hold other reviews left by other users.

Reviews - Previous reviews that have been posted by users.

 

2.3 Control Objects

User Controller - Coordinates all data that relates to the user.

User Interface - Coordinates all data related to leaving a review, and all other actions that can take place.

 

2.4 Boundary Objects

Registration View - The registration page presents new users an option to create a brand new account. Users will be able to create an account using any email address (from any domain) and a password that meets the complexity requirements. If a user enters an email that has already been registered then, an error message will appear to indicate this email is already in the records. As well, if a password is typed in and it does not meet the complexity requirements then, an error message will appear indicating a stronger password must be chosen.  

Login View - The login page will be used to store the user credentials and identify the particular user. This will require two pieces of information from the user, their User Identification and password. If the user enters credentials that are not correct (they do not associate with an existing user record) then, an error message will be displayed asking users to enter their information once again. If the user forgets their password, they can be sent a one time 6 digit code to their email in order to reset their password.

Search View - The search option will allow the user to search the entire website for any Laurier course. The user must input the exact course code or the full name of the course they want to search. The course that matches the search will be shown in a drop down menu, from there the user can select that course and they will be taken to the course page. If a user enters an invalid input (e.g. a non-Laurier course) then, an error message will appear saying “No course was found”.

Post View - The post function is used to create a new Laurier course review. The user will have a chance to create a post once they are on a specific course page. Once they are on a specific course page, they can click the “new post” button.
